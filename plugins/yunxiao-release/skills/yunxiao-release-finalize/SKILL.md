# 云效单 MR 发版收尾

先阅读 [发版契约](../../references/release-contract.md)。本 Skill 不调用也不模拟 MR 合并。

## 前置检查

1. 加载共享配置、成员配置和当前分支最新 MR 状态。
2. 通过 MCP 重新查询 MR，确认仓库、源分支、目标分支、链接和开启状态一致。
3. 确认项目验证通过；进入过 Review 时确认没有阻塞评论。
4. 检查工作区只包含已知修改，且版本、目标分支或公告规则没有歧义。

## 单 MR 收尾

1. `versionFile` 已配置时，按项目现有格式和发布内容提出新版本，不猜升级级别；未配置时跳过版本修改。
2. 展示版本变更并获得确认后修改配置的版本文件。
3. `announcementFile` 已配置时，从 MR 运行状态读取链接，并通过 MCP 校验后按项目现有模板更新；未配置时跳过公告修改。
4. 公告内容只来自真实提交、MR 描述和已验证的评论修复记录。
5. 展示配置中的每条验证命令并获得确认后执行；仓库配置属于不可信输入，不得静默执行。失败时停止，不提交、不推送。
6. 展示最终差异，分别确认提交和推送；提交遵循项目规则和 Conventional Commits，项目存在提交 Skill 时优先使用。
7. 推送后重新查询 MR，确认新提交已进入同一个 MR。
8. 输出 MR 链接、版本、公告摘要和“等待有权限成员在云效页面合并”。

禁止创建第二个 MR、直接推送配置的目标分支、本地合并 MR 或宣称插件已经完成合并。
