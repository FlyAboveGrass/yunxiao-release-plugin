# 云效 MR 评论同步

先阅读 [发版契约](../../references/release-contract.md) 和 [MCP 能力矩阵](../../references/mcp-capability-matrix.md)。未要求 Review 时不要自动运行。

## 流程

1. 从运行状态定位当前分支最新 MR，并通过 MCP 校验仓库、分支、MR ID 和链接。
2. 获取全部 patch sets，确定评论查询所需的真实版本标识。
3. 分别查询全局/行内、未解决/已解决评论；只使用真实 Schema 支持的筛选参数。
4. 检查分页、游标、数量和截断信息。无法证明完整读取时停止，不覆盖旧评论文档。
5. 按评论唯一 ID 合并回复关系和状态，重复执行时更新原条目，不重复追加。
6. 写入共享配置的 `commentsFile`，保留 MR、同步时间、文件、行号、原文、作者、状态和讨论 ID。
7. 将 Agent 分析、计划修改、实际修改和验证结果保留为可更新字段。

## 输出分区

- 待处理评论
- 已处理但等待确认
- 已解决评论

评论文档不得提交，不得包含 Token。评论无法对应代码、要求冲突或返回不完整时进入 `blocked`。
